# OctoAcme — Cross-Functional Responsibility Matrix (RACI)

## Purpose
Clarify role accountability across the delivery lifecycle so ownership is explicit and handoffs are consistent.

## RACI key
- **R** = Responsible (does the work)
- **A** = Accountable (owns the outcome/decision)
- **C** = Consulted (provides input)
- **I** = Informed (kept updated)

## Lifecycle matrix

| Lifecycle phase | Developers | Product Managers | Project Managers | UX Designers | Technical Writers | QA Leads | Scrum Masters | Business Analysts |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| Initiation | C | A | R | C | I | I | I | R |
| Planning | R | A | R | C | C | C | C | R |
| Execution | R | A | R | R | C | R | A | C |
| Release | R | A | A | C | R | R | C | C |
| Retrospective | R | C | A | C | I | R | R | C |

## Notes for use
- Assign named individuals for each role at kickoff.
- If one person covers multiple roles, retain the role-based RACI to preserve clarity.
- Review this matrix during planning and update when team composition changes.
