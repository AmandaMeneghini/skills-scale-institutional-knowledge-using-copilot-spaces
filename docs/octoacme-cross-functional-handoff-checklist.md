# OctoAcme — Cross-Functional Handoff Checklist

## Purpose
Provide a lightweight checklist to reduce handoff gaps between design, development, QA, and release activities.

## 1) Design-to-Development handoff
- [ ] UX design artifacts are linked (flows, wireframes, final states)
- [ ] Accessibility expectations are documented
- [ ] Edge cases and error states are captured
- [ ] Acceptance criteria are reviewed with Product Manager and Developers
- [ ] Technical constraints/trade-offs are acknowledged by UX + Development
- [ ] Open design questions have owners and due dates

## 2) Development-to-QA handoff
- [ ] Scope delivered matches approved acceptance criteria
- [ ] Test environment and data setup instructions are provided
- [ ] Unit/integration test status is shared
- [ ] Known limitations and deferred defects are documented
- [ ] Feature flags/config dependencies are listed
- [ ] Demo or walkthrough completed with QA Lead

## 3) Release readiness handoff
- [ ] QA sign-off and critical defect status are recorded
- [ ] Release notes are drafted/reviewed (Product + Technical Writer)
- [ ] Rollback and incident response steps are documented
- [ ] Monitoring/alerts and post-deploy verification steps are defined
- [ ] Stakeholder communication plan is ready (owner, audience, timing)
- [ ] Go/no-go decision is captured with accountable owner(s)

## Ownership guidance
- **Project Manager:** confirms checklist completion status and escalates blockers.
- **Product Manager:** confirms scope and acceptance outcomes for release decisions.
- **Developers + QA Lead + UX Designer + Technical Writer + Business Analyst:** complete section-specific inputs and document unresolved risks.
- **Scrum Master:** supports completion during ceremonies and tracks blocked items to closure.
